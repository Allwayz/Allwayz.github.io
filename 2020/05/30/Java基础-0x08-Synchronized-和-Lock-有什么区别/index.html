Synchronized 和 Lock 有什么区别

原始构成

  • synchronized是Java关键字 属于JVM层面

    Monitorenter（底层通过monitor对象来完成，其实wait/notify等方法也依赖于monitor对象，只有在同步块或方法中才能调用wait/notify等方法

    现在情况是有一个同步代码块synchronized(new Object){}。

    将.java文件使用java -c 编译成.class文件之后查看java字节码可以看出。

    在同步代码块中，进入时使用的是monitorenter方法，执行完毕之后调用monitorexit方法，而且调用了两次，一次是正常退出，一次是异常退出。所以synchronized可以保证不会出现死锁。

  • Lock是具体的类(java.util.concurrent.locks.Lock)。是API层面的锁。

使用方法

  • synchronized不需要开发者去手动释放锁，当synchronized代码执行完成后，系统会自动让线程释放对锁的占用

  • lock则需要开发者手动释放锁，若没有主动释放锁，就可能会出现思索先行

    需要lock()&unlock()方法配合try/finally代码块来完成

等待是否中断

  • synchronized不可中断，除非抛出异常或者正常运行完成
  • ReentrantLock可以中断
    • 设置超时方法 tryLock(long timeout, TimeUnit unit)
    • lockInterruptibly()放在代码块中，调用interrupt()方法可中断

加锁是否公平

  • synchronized 非公平锁
  • ReentrantLock两者都可以，默认是非公平锁，构造方法中可以穿入boolean值，true为公平锁,false为非公平锁。

锁绑定多个条件Condition

  • synchronizednot support
  • ReentrantLock用来实现分组唤醒需要唤醒的线程，可以精确唤醒。而不是像synchronized要么随机唤醒一个，要呢全部唤醒。
