Java基础-0x0d:GC垃圾回收初探

发表于 2020-06-02
字数统计:

JVM

JVM体系结构概览

对于堆内存而言：

有伊甸园区，幸存者0区和幸存者1区。

GC的作用与在方法区和堆。

常见的垃圾回收算法

引用计数

有对象引用的话，就+1。没有对象引用就-1。到0为止

缺点：

  • 每次对对象赋值时均要维护引用计数器，且计数器本身也有一定的消耗；
  • 较难处理循环引用

JVM的实现一般不采用这种方式

复制

复制算法在年轻代使用。

口诀：复制之后有交换，谁空谁是To

其实就是Survivor0和Survivor1区中的复制，交换，清空。

  1. eden、SurvivorFrom复制到ServivorTo，年龄+1

    首先，当Eden区满的时候会触发第一次GC，把还活着的对象拷贝到SurvivorFrom区，当Eden区再次触发GC的时候会扫描Eden区和From区域，对这两个区域进行垃圾回收，经过这次回收后还存活的对象，则直接复制到To区域（如果有对象的年龄已经达到了老年的标准，则复制到老年代区），同时把这些对象的年龄+1.

  2. 清空eden、SurvivorFrom

    然后，清空Eden和SurvivorFrom中的对象，也即复制之后有交换，谁空谁是To

  3. SurvivorTo和SurvivorFrom互换

    最后，SurvivorTo和ServivorFrom互换，原ServivorTo成为下一次GC时的SruvivorFrom区。部分对象会在From区和To区域中来回复制，如此交换15次（由JVM参数MaxTenuringThreshould决定，这个参数默认是15），最终如果还是存活，就放入老年代。

优点：没有产生内存碎片，因为是整体复的。

缺点：浪费空间，一些大的对象复制起来会耗时。

标记清除（Mark-Sweep）

先标记 –> 后清除：算法分为标记和清除两个阶段，先标记出要回收的对象，然后统一回收这些对象。

优点：对象没有大面积复制，节约了内存空间

缺点：容易产生内存碎片

  • 标记整理

    原理：

    1. 标记（mark）

      与标记清除算法一样

    2. 压缩（Compact）

      再次扫描，并往一段滑动存活对象

    优点：没有内存碎片

    缺点：需要移动对象的成本

复制多用与年轻代，标记清除和标记整理多用于老生代。

标记整理既不会产生内存碎片，也不浪费空间，但是耗时严重。

JVM垃圾回收的时候如何确定垃圾

什么是垃圾？

简单的说就是内存中不再被使用到的空间就是垃圾

如何判断一个对象是否可以被回收

引用计数法 ==不再使用==

  • 枚举跟节点做可达性分析（跟搜索历经）

    为了解决引用计数法的循环引用的问题，Java使用了可达性分析的方法

    所谓，GC roots 或者说 tracing GC的根集合 就是==一组必须活跃的引用==。

    ==基本思路就是通过一斜裂名为GC Root的对象作为起始点==，从这个被称为GC Roots的对象开始向下搜索，如果一个对象到GC Roots没有任何引用链相连，则说明此对象不可用。也即给定一个集合的引用作为根出发，通过引用关系遍历对象，能被遍历到的（可达的）对象就被判定为存活；没有被遍历到的（不可达）就自然被判定为死亡。

哪些对象可以作为GC Roots的对象？

  • 虚拟机栈（栈帧中的局部变量区，也叫局部变量表）中引用的对象
  • 方法去中的类静态属性引用的对象
  • 方法去中常量引用的对象
  • 本地方法栈中JNI（Native方法）引用的对象
